Git & GitHub
  • If git is not recognized: install Git for Windows from git-scm.com and restart the terminal; use PATH option that exposes Git to the command line.
  • Create the empty repo on GitHub first (no README if you already have local files). Otherwise git push → “Repository not found” / 404.
  • Set commit identity (two dashes: --global):
Git user (required before commit) 
git config --global user.name "Your Name"
git config --global user.email "your-email@example.com"

Push with upstream (no space: -u, not - u):

First push 
git remote add origin https://github.com/YOURUSER/YOURREPO.git
git branch -M main
git push -u origin main

Fix wrong remote: git remote set-url origin <correct-url>

Vercel
  • Push code to GitHub.
  • vercel.com → Import project → click Install to connect the GitHub app.
  • Framework: Next.js; root = repo root.
  • Deploy. No “app user accounts” needed for hosting — only your GitHub + Vercel logins.
PWA checks (Chrome)
  • DevTools → Application → Manifest (manifest loads).
  • Application → Service workers → sw.js active after load (may need refresh).
  • Network → Offline → reload: shell should still load once cached.
  • Strange runtime.lastError in normal window but clean in Incognito → often a browser extension, not the app.
  • /_next/static/chunks/... 500 → stop server, npm run build, restart, clear site data, hard refresh.
TypeScript tip

Error: 'React' refers to a UMD global... — often from React.ReactNode without importing React. Use:

layout.tsx pattern 
import type { ReactNode } from "react";
// children: ReactNode
